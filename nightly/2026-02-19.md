# 2026-02-19

## MVP Dashboard UX Overhaul

The single-scroll client detail page was replaced with a structured, tabbed layout designed for daily ops. Staff can now understand a client's current state within 10 seconds of opening their profile.

### Sticky Header + Client Snapshot

- A compact sticky bar stays visible while scrolling, showing the client name, internal ID, priority, contact info, and quick action buttons.
- A 4-card snapshot grid sits at the top of every client view: Case Status (with a toggle between "most urgent", "all statuses", and "primary contract"), Next Deadline, Case Type/Relief, and Last Action.
- All snapshot data is derived from existing API responses — no new queries needed.

### Tab Navigation

- Four tabs replace the long scroll: Overview, Documents & Notices, Appointments, and Relations.
- Overview shows contracts, active case boards, and the enhanced timeline.
- Documents & Notices pulls from RFE, Originals/Cards/Notices, NVC Notices, and Address Changes boards.
- Relations shows linked board items (e.g., a motion linked to its court case) from the item_relationships table.
- Tabs are reflected in the URL hash (`#client/{id}/documents`) so deep linking and back-button work.

### Timeline Enhancements

- Filter chips let staff narrow the timeline to Notes, Documents, Notices/RFEs, or Appointments.
- A "Last 30 days" quick toggle cuts through long histories.
- "Load more" pagination (30 entries per page) replaces the old 5-date-group limit.
- Each entry now has a color-coded event type badge (Note, Document, Appointment, Reply).

### Action Buttons

- "Copy Client Summary" generates a plain-text summary and copies it to clipboard.
- "Generate Document" and "Open in Monday" are placeholder buttons (disabled, labeled "Coming soon") ready for future integration.
- "View Relations" jumps directly to the Relations tab.

### Backend

- New API endpoint: `GET /api/clients/:id/relationships` returns linked board items with resolved names and statuses.
- Updates endpoint now supports `offset` for proper pagination.
- PII fixture imports made optional so the seeder works whether or not real profile files are present on disk.

- 9 new React components, 3 refactored, 6 backend files modified.
