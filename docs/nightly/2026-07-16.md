## SharePoint file browser — it works

IT approved the Microsoft access that was the last blocker, and we tested the feature signed in with a real account. It works:

- The **Documents** tab lists a client's SharePoint folders (e-file, consult, etc.), you browse into them, and open or download any file — without leaving the dashboard.
- **Uploading** into a client's folder works too.
- Access is **per person**: each staff member only ever sees the files SharePoint already lets them open. The dashboard grants nobody extra reach.

**New today: document preview.** Clicking a file now opens it **inside the dashboard** — PDFs and Word/Excel/PowerPoint render in place, images show directly. Anything that can't be previewed (e.g. a zip) falls back to Download / Open in SharePoint. Reusing the same Microsoft access, so no new permission was needed.

**Still to do:** this is verified locally against the real data. The last steps are a final review and deploying it to the server — at which point the **Documents tab changes**: it becomes the file browser, and the four notice lists it used to show (RFEs, notices, etc.) are retired from the client page (an agreed trade-off).

## Two production bugs fixed (ready to deploy)

Both were spotted while using the live server.

**1. Watchlist and "recently viewed" showed gibberish.** Pinned clients and recent history displayed a long code instead of the client's name, and clicking one said "Client not found."

The cause: the app was remembering clients by an internal id that **the nightly sync regenerates every night**. So every stored pin pointed at nothing by the next morning. It now remembers clients by their **stable Monday.com id**, which never changes. (Note: because the old saved pins can't be recovered, they're cleared on deploy — everyone re-pins once.)

**2. The "Monday.com" link on a client went nowhere useful.** It used a generic address instead of the firm's own Monday.com address (`scaltheclinic.monday.com`). Fixed — the link now opens the correct item.

Both are merged and the deployable image is built; they go live on the next server deploy.
