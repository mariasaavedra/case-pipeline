## Dashboard now fits tablets and phones

Until today, the dashboard was built for a full desktop screen. Opening it on a tablet or a smaller laptop window would either squeeze everything together or hide the navigation entirely. That's fixed.

### What changed by screen size

**Tablets and split-screen windows (641–1024px wide)**
- The left-hand navigation bar now stays visible at all times as a slim 60px icon-only strip.
- Icons still work — clicking any of them navigates to that page as normal.
- The hamburger menu button is hidden (it isn't needed since the nav is always on screen).
- Content fills the remaining space correctly without overlapping the nav.

**Phones (640px and narrower)**
- The hamburger menu button returns and the drawer works as before.
- On a client's profile page, the client's full name now gets its own line at the top. The priority badge (Urgent / High / Medium / Low) and the action buttons wrap beneath it rather than squeezing the name.
- The tab strip at the bottom of a client page fades at the right edge to signal that it can be swiped or scrolled.

**Desktop (wider than 1024px)**
- No change. The full 220px sidebar works exactly as before, including the manual collapse toggle.

### Behind the scenes

Four files were updated and one new file was created — all inside the web dashboard. No data, no database, and no Monday.com connection were touched. Type-checking confirms zero errors introduced.

## New planning and browser tools added

To support interface planning and ongoing development, three tool integrations were wired into the project:

- **Playwright** — allows the development environment to automatically open the dashboard in a browser, take screenshots, and click through pages to confirm that changes look right at different screen sizes. This means visual checks can be done by the tool rather than manually.
- **GitHub** — connects the development environment directly to this project's code repository on GitHub, making it easier to track what was changed, reference issues, and manage code history. *(Requires a one-time token to be entered — see developer notes.)*
- **Notion** — already connected via the existing claude.ai account; available for planning and documentation. *(Requires a one-time sign-in to authorize.)*
