# Daily Update — June 24, 2026

*Written for non-technical stakeholders. Plain language, no jargon.*

## In one line

The dashboard's home page cards were corrected to show the **right data from Monday.com**, and a bug that crashed the Contracts tab for every client was fixed.

---

## Contracts tab no longer crashes

- Opening the Contracts tab on any client page was throwing an error and showing nothing.
- The cause: some contracts in Monday.com have no dollar amount entered yet, and the dashboard was not prepared to handle a blank value.
- Fixed — the tab now loads correctly, showing a dash ("—") wherever a contract amount hasn't been filled in yet.

## Home page cards now pull from the correct Monday.com groups

Previously, the three contract-related cards on the home page used status labels to decide what to show. This was unreliable because the same status can appear in multiple groups. Each card is now tied directly to the **Monday.com group** it is supposed to reflect.

### Open Forms
- Now shows exactly the items inside the **"Open Forms" group** on the Open Forms board.
- The board was re-synced today so any changes made in Monday.com this session are reflected.

### Pending Contracts
- Now shows exactly the items inside the **"Pending Fee Ks" group** on the Fee Ks board.
- Previously it was showing 274 items using a loose status filter. The correct count is now 38.

### Prescheduling *(formerly "Paid Fee Ks")*
- Renamed to **Prescheduling** across the entire dashboard.
- Now shows exactly the items inside the **"Paid Fee Ks" group** on the Fee Ks board.

## Behind the scenes

- The database was updated to store which Monday.com **group** each contract belongs to, so future filters can use it without re-syncing.
- The Fee Ks board was re-synced to populate that information (1,405 contracts updated).
