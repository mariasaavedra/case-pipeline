# Nightly Log — 2026-05-26

## Summary

Full planning and implementation session. The app gained a new `/active-cases` page — a swim-lane grid showing every active open form case, organized by paralegal row and urgency column.

---

## 1. Codebase Audit

Reviewed all current feature docs, source files, and the query/schema layer. Key findings:

- **`next_date` not backfilled for open forms** — the v3 migration's `COALESCE` list covered court cases (`x_next_hearing_date`), motions (`next_hearing_date`), and general items (`due_date`), but not open forms (`target_date`). `BoardItemFactory.NEXT_DATE_KEY` correctly handles new inserts, but any DB upgraded in-place had null `next_date` for all open forms.
- **`paralegals` not a promoted column** — assignee lives in `column_values` JSON only. Without a first-class column, queries can't group or index on it.
- **Seeded open forms were unrealistic** — all target dates 14–90 days future (overdue/critical columns always empty), no paralegal assignment (all "Unassigned"), no terminal groups (all items would appear as active).

## 5. Query Layer

**New file:** `libs/query/src/active-cases.ts`  
**New types:** `Urgency`, `ActiveCase`, `ActiveCasesAssignee`, `ActiveCasesResult`  
**Exported from:** `libs/query/src/index.ts`

Single flat SQL query joining `board_items` + `profiles`, filtered to `board_key = '_cd_open_forms'` and `group_title IN ('Open Forms', 'Court Forms')`. TypeScript post-processing handles urgency bucketing, `isCourtCase` derivation, grouping by assignee, and sorting (assignees alphabetically / Unassigned last; cases overdue → none within each row).

**Urgency thresholds:**

| Bucket | Condition |
|---|---|
| `overdue` | diff < 0 |
| `critical` | 0 ≤ diff ≤ 3 |
| `soon` | 4 ≤ diff ≤ 7 |
| `later` | diff ≥ 8 |
| `none` | targetDate is null |

**New test file:** `libs/query/src/active-cases.test.ts` — 10 tests covering: empty DB, grouping by paralegal, Unassigned row, alphabetical sort, all 5 urgency buckets, `daysUntilTarget` sign, `isCourtCase`, terminal group exclusion, other-board exclusion, and within-assignee urgency ordering.
