## ⚠️ "North Pole" cases are being forgotten

We looked into the **"Send to North Pole"** status, which the team uses to temporarily park a case (for example, when a client asks for a week to find a document). The case is still pending — it just shouldn't clutter the active list.

There's already a **return date** field for this. Here's what the 8 parked cases actually look like:

- **3** are properly parked, with a future return date.
- **4** have a return date that **already passed** — by 8 days, 12 days, 142 days, and **320 days**. They should have come back long ago. Nobody noticed.
- **1** has **no return date at all**.

Four of them also have an overdue target date — one by 288 days.

**The point:** "temporarily hidden" has been quietly turning into "permanently forgotten." The good news is these cases are still *visible* today, so nothing is lost yet. But if we simply hid them (the obvious way to build this), those five would vanish and the problem would get worse.

**The plan** is to make it self-correcting: a case is hidden **only while its return date is still in the future**, and it reappears on its own the day the date passes. A real deadline still raises an alert even while parked — parking a case must never silence a legal deadline. Alongside that, a small Monday.com automation would set a default return date (two weeks) whenever a case is sent to North Pole, so a case can't be parked without a way back.
