# Daily Update — July 14, 2026

*Written for non-technical stakeholders. Plain language, no jargon.*

## In one line

The user system was significantly strengthened — Monday.com tokens are now encrypted inside the live database, each person's dashboard remembers their settings, paralegals get a personal "My Cases" view, and every sensitive action is now recorded in an audit log. Along the way we found and fixed a hidden bug that had been leaving case assignments blank.

---

## Monday.com tokens are now encrypted inside the database

Yesterday's update encrypted the database *backups*. Today we closed the loop: the Monday.com access tokens stored **inside the live account database** are now encrypted too (AES-256). Previously they sat in plain text — anyone who obtained the database file could have used them to act as that staff member on Monday.com. Now they're unreadable without the encryption key, which is kept separately and never stored with the data.

This was the "optional future item" noted at the end of yesterday's update — it's done.

---

## The dashboard now remembers each person

Until now, everyone saw exactly the same dashboard with no memory of their choices. Now each person has their own settings that **follow them across devices** (they're saved to their account, not just one browser):

- **Light / dark / system theme.**
- **Which page they land on** when they sign in.
- Display density and other view preferences.

---

## "My Cases" — every paralegal sees their own workload

There's a new **My Cases** page. Each paralegal links their account to their name on the boards (in Settings → "I am"), and the page then shows just *their* open cases, most urgent first. Administrators can also set this link for staff from the admin panel, and can turn a person's access on or off without deleting their history.

---

## Quick access: watchlist, recently viewed, and saved searches

Several touches to make daily work faster:

- **Watchlist** — pin the clients you're actively working, and reach them in one click from the home page.
- **Recently viewed** — the last clients you opened are always a click away.
- **Saved searches** — save a set of filters on the Clients page (e.g. "urgent, this attorney") and re-apply it instantly.

---

## A full record of sensitive actions (audit log)

In a law practice, "who did what" matters. The system now keeps an **audit log** of sensitive actions — changing someone's role, posting a note to Monday.com, editing a staff profile, or changing board settings. Administrators can review it from Settings.

---

## A hidden bug: case assignments were coming through blank

While building "My Cases," we discovered that the **paralegal assigned to each case was not being imported** from Monday.com — the field had been silently empty for some time. This didn't only affect the new feature; it also meant the **Active Cases board was showing work as "Unassigned"** even when a paralegal was assigned.

The cause was a naming mismatch in how the data was read from Monday.com. It's now fixed, and we recovered the existing assignments **without needing to re-pull everything from Monday.com** — the information was already stored and just needed to be re-read. On the local copy, that restored paralegal assignments across ~1,600 cases (17 paralegals).

---

## The automated test system is green again

The project's automated checks (which run on every change) had been failing on every commit for weeks — they required a data folder that doesn't exist in that clean environment. That's fixed, so the safety net that catches mistakes before they ship is working again. All existing tests pass.
