# Daily Update — June 23, 2026

*Written for non-technical stakeholders. Plain language, no jargon.*

## In one line

The dashboard now requires a **secure sign-in**, has a **Settings page**, and will automatically pull in **all client notes from Monday.com** going forward.

---

## Sign-in is now required

- The dashboard is no longer open to anyone who knows the URL.
- Staff sign in using their **existing firm Microsoft account** — the same one used for Outlook and SharePoint. No new passwords to remember.
- The first person to sign in was automatically made the administrator.
- Two access levels:
  - **Admin** — can manage who has access and what role they have.
  - **User** — full access to all client data, no user management.
- A sign-in page greets anyone who isn't logged in.
- Signing out is available from the sidebar at any time.

## Settings page

- Every user now has a **Settings page** (gear icon in the sidebar).
- Personal preferences that can be changed:
  - **Theme** — Light or Dark appearance.
  - **Default page** — which page opens after signing in.
  - **Sidebar** — start collapsed (icon-only) or expanded.
  - **Date format** — choose how dates appear across the dashboard.
- **Admins only** — a User Management section inside Settings shows the full list of staff who have signed in, when they last logged in, and lets an admin promote or demote their role with one click.

## Automatic data refresh

- The dashboard will now automatically pull fresh data from Monday.com **every night at midnight**, so the team always sees up-to-date information in the morning without anyone having to run anything manually.
- This only works while the server machine is on. A future step will move this to a dedicated server so it runs reliably every night regardless.

## Client notes are now being imported (in progress today)

- We confirmed that Monday.com's comment system (the Updates tab on every item) is accessible and readable.
- The sync engine has been updated to pull in **every note and comment** written on every client profile and every linked case, contract, court case, appointment, and more.
- All notes for a client — regardless of which board they were written on — will appear together in one unified timeline on that client's page.
- Notes are tagged with which board they came from (e.g. "from Court Cases") so context is never lost.
- **Emails & Activities** (the email log inside Monday) cannot be imported — Monday does not make that data available through their API. Everything else (written comments and replies) is included.
- The first full import with notes is running as of today and will complete shortly.

## What's next

- Verify the notes timeline looks correct in the dashboard once today's import finishes.
- Begin work on per-user saved views and preferences stored on the server.
- Plan the move to an always-on server so nightly sync is guaranteed.

## Privacy & safety

- Sign-in is handled entirely by Microsoft — no passwords are stored in the dashboard.
- Real client data remains on a local machine and is never published online.
